# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, please contact us at **security@example.com**. You may also use our [security contact form](https://example.com/security-contact) for confidential reporting. If you wish to encrypt your report, our PGP public key is available [here](https://example.com/pgp-key).

**Please do not include sensitive details in public issues.** Reference UTCS record IDs where relevant.

## Expected Response Time

We aim to respond to security reports within **3 business days**. After initial contact, we will keep you informed of the progress as we investigate and address the issue.

## Scope

The following are considered security issues:
- Remote code execution
- Privilege escalation
- Authentication or authorization bypass
- Data leaks or exposure of sensitive information
- Denial of service (DoS) vulnerabilities

The following are **not** considered security issues:
- Vulnerabilities requiring physical access to the device
- Social engineering attacks
- Outdated dependencies with no exploitable impact

## Supported Versions

We currently support the following versions for security updates:
- v2.x (latest)
- v1.5.x (LTS)

Older versions are **not** supported and will not receive security updates.

## Bug Bounty

At this time, we **do not** operate a public bug bounty program. However, we recognize and appreciate responsible disclosure and may offer recognition or rewards at our discretion.

Thank you for helping us keep our project and users safe!
